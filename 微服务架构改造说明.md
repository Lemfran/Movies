| 原组件 | 状态 | 替代方案 | 说明 |
|--------|------|---------|------|
| Zuul | 已弃用 | Spring Cloud Gateway | 等效的路由转发 + 过滤器链功能（pre/post/route类型） |
| Ribbon | 已弃用 | Spring Cloud LoadBalancer | 等效的客户端负载均衡功能 |
| Hystrix | 已停止维护 | Resilience4j | 新一代容错框架，功能更强大 |

## 五、微服务网关（Spring Cloud Gateway）

### 路由规则

```yaml
# Gateway 路由配置 (application.yml)
/api/**      -->  lb://movies-service              # 电影REST API
/rpc/**      -->  lb://movies-service              # 内部RPC接口
/auth/**     -->  lb://movies-service              # JWT认证接口（白名单）
/consumer/** -->  lb://movies-consumer             # 消费者页面（白名单）
/recommend/** --> lb://recommendation-service      # 推荐服务
```

### 过滤器链（对应 Zuul 过滤器）

| 过滤器 | 类型 | 顺序 | 功能 |
|--------|------|------|------|
| `LoggingFilter` | GlobalFilter (post) | order=0 | 记录所有请求的方法、URI、IP、耗时 |
| `AuthFilter` | GlobalFilter (pre) | order=1 | JWT验证、白名单放行、用户信息注入 |

### JWT 认证过滤器流程

1. 请求到达 Gateway (8080)
2. `LoggingFilter` 记录请求日志
3. `AuthFilter` 检查路径是否在白名单
   - 白名单路径：`/auth/login`, `/auth/register`, `/consumer`, `/eureka`, `/actuator`, `/api/info`
   - 白名单路径直接放行
4. 非白名单路径：从 Header (`Authorization: Bearer <token>`) 或 Cookie 提取 JWT
5. 验证 Token 有效性
   - 无效/过期 → 返回 401 Unauthorized
   - 有效 → 将 `X-User-Id`, `X-Username` 注入请求头 → 转发给下游服务

## 六、统一认证与授权（JWT）

### 认证流程

```
用户                    Gateway                    movies-service
 |                        |                              |
 |---POST /auth/login---->|                              |
 |  username+password     |                              |
 |                        |---转发到 /auth/login-------->|
 |                        |                              |---查询数据库验证
 |                        |                              |---生成JWT Token
 |                        |<---返回 {token, ...}---------|
 |<---返回JWT Token-------|                              |
 |                        |                              |
 |---后续请求携带Token---->|                              |
 |  Authorization: Bearer |                              |
 |                        |---验证Token有效-------------|
 |                        |---注入X-User-Id/X-Username-->|
 |                        |                              |---执行业务逻辑
```

### 新增接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/auth/login` | POST | 用户名+密码登录，返回JWT Token |
| `/auth/register` | POST | 注册新用户，返回JWT Token |
| `/auth/verify` | GET | 验证Token有效性 |

### 关键代码

- `movies-service/src/main/java/edu/cuit/yingpingsxitong/util/JwtUtil.java` — Token 生成/验证
- `movies-service/src/main/java/edu/cuit/yingpingsxitong/Controller/AuthController.java` — 认证接口
- `movies-service/src/main/java/edu/cuit/yingpingsxitong/config/SecurityConfig.java` — Security配置
- `gateway-service/src/main/java/edu/cuit/gateway/filter/AuthFilter.java` — 网关统一拦截

> **注意**：原有基于 Session 的登录功能（`IndexController.login()`、`UserController`）完全保留，用户仍可直接访问 `http://localhost:8081` 使用原有 Thymeleaf 页面。JWT 认证是通过 Gateway 提供的新的统一认证入口。

## 七、容错处理（Resilience4j）

### 设计要点：避免同类自调用

**重要**：Spring AOP 代理只能拦截外部方法调用，**不能拦截同类方法之间的自调用**。因此将 HTTP 调用从 `RecommendationService` 抽离到独立的 `MovieClientService` 中：

```java
@Service
public class MovieClientService {

    @CircuitBreaker(name = "movies-service", fallbackMethod = "fetchAllMoviesFallback")
    @Retry(name = "movies-service")
    public List<Map<String, Object>> fetchAllMovies() {
        // 调用 movies-service /api/movies
    }

    public List<Map<String, Object>> fetchAllMoviesFallback(Exception ex) {
        // 降级：返回 "服务暂时不可用" 提示数据
    }
}
```

```java
@Service
public class RecommendationService {
    @Autowired
    private MovieClientService movieClientService;  // 注入外部服务，AOP可拦截

    public List<Map<String, Object>> getHotRecommendations(int limit) {
        return movieClientService.fetchAllMovies().stream()...;
    }
}
```

### 配置参数

```properties
# 熔断器配置
resilience4j.circuitbreaker.instances.movies-service.registerHealthIndicator=true
resilience4j.circuitbreaker.instances.movies-service.slidingWindowSize=5
resilience4j.circuitbreaker.instances.movies-service.permittedNumberOfCallsInHalfOpenState=3
resilience4j.circuitbreaker.instances.movies-service.waitDurationInOpenState=10s
resilience4j.circuitbreaker.instances.movies-service.failureRateThreshold=50

# 重试配置
resilience4j.retry.instances.movies-service.maxAttempts=3
resilience4j.retry.instances.movies-service.waitDuration=1s
```

### 效果验证

```
[熔断降级] movies-service 调用失败，返回降级数据。原因: I/O error on GET request...
[熔断降级] movies-service 调用失败，返回降级数据。原因: I/O error on GET request...
[熔断降级] movies-service 调用失败，返回降级数据。原因: CircuitBreaker 'movies-service' is OPEN and does not permit further calls
```

- 失败率超过 50% 时，熔断器 OPEN
- 10 秒内所有请求直接走降级方法
- 10 秒后进入半开状态，尝试恢复

## 八、负载均衡（LoadBalancer）

### 默认使用 Spring Cloud LoadBalancer（Ribbon 的替代方案）

- `movies-consumer` 中配置了 `@LoadBalanced RestTemplate`
- 通过服务名 `http://movies-service` 发起调用
- LoadBalancer 自动从 Eureka 获取实例列表并分配请求

### 自定义策略演示

```java
@Configuration
public class CustomLoadBalancerConfig {
    @Bean
    public ReactorLoadBalancer<ServiceInstance> randomLoadBalancer(...) {
        return new RandomLoadBalancer(...);  // 随机策略（替代默认轮询）
    }
}
```

## 九、配置中心（Config Server）

### 技术注意

Config Server 使用 `native` 模式（本地文件系统）而非 Git，以兼容含中文字符的路径：

```properties
spring.profiles.active=native
spring.cloud.config.server.native.search-locations=file:///Users/fanjinchen/工程实践/Movies/Movies/config-repo
```

同时 `movies-service` 的 `application.properties` 中保留了数据库等核心配置的本地 fallback，确保 Config Server 不可用时服务仍能启动。

## 十三、改造中遇到的关键问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| Config Server 中文路径报错 | Git URI 编码问题 | 改用 `native` 模式 |
| movies-service 数据库配置获取失败 | Config Server 未正确启动 | `application.properties` 中添加本地 fallback |
| movies-service 启动失败 | Security 依赖引入后缺少配置 | 添加 `SecurityConfig` 关闭 CSRF 并放行所有请求 |
| Resilience4j 熔断不生效 | `@CircuitBreaker` 在同类方法自调用上，AOP 无法拦截 | 将 HTTP 调用抽离到独立的 `MovieClientService` |
| Gateway CORS 报错 | `allowCredentials: true` 不能与 `allowedOrigins: "*"` 同时使用 | 改为 `allowCredentials: false` |
| JWT `userId` 类型不匹配 | Gateway 用 `String.class` 读取 Integer 类型的 claim | 改为 `claims.get("userId").toString()` |
| 新注册用户无法登录 | 默认 `permission=false` | MySQL 中手动 `UPDATE users SET permission=1` |
| `/consumer/**` 被 Gateway 拦截 | 不在白名单中 | 添加到 `WHITE_LIST` |
